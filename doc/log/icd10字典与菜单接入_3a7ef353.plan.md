# ICD-10字典精简方案与菜单接入计划

## 目标与范围

- 新增一张精简版 ICD-10 疾病诊断字典表（仅保留核心字段：编码、名称、拼音首字母、状态、排序、时间戳）。
- 在标准数据管理页面新增“疾病诊断(ICD-10)”左侧菜单，并提供与现有模块一致的单列表页。
- 使用 PostgreSQL MCP 执行建表 SQL（含注释、唯一约束、常用索引）。

## 精简表结构（拟定）

- 表名：`emr_icd10_diagnosis`
- 字段：
  - `id BIGSERIAL PRIMARY KEY`
  - `diag_code VARCHAR(32) NOT NULL`（疾病编码，唯一）
  - `diag_name VARCHAR(256) NOT NULL`（疾病名称）
  - `pinyin_initials VARCHAR(64) NOT NULL`（拼音首字母，用于快速检索）
  - `status SMALLINT DEFAULT 1`（1启用/0停用）
  - `sort_no INTEGER DEFAULT 0`（排序）
  - `create_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP`
  - `update_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP`
- 约束与索引：
  - 唯一索引：`uk_emr_icd10_diag_code (diag_code)`
  - 检查约束：`status IN (0,1)`
  - 普通索引：`idx_emr_icd10_diag_name (diag_name)`、`idx_emr_icd10_pinyin_initials (pinyin_initials)`

## 后端配置接入（APIJSON可读写）

- 对齐现有别名风格，确保前端可用 `EmrIcd10Diagnosis`（或统一别名）发起 `/apijson/get` 列表查询。
