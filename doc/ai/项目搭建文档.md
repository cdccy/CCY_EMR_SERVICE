## 1. 项目概述

### 1.1 项目背景

这是一个全新搭建的**电子病历管理系统 (EMR - Electronic Medical Record)** 后端服务，采用现代化的技术栈和架构设计，为医疗机构提供安全、可靠、高效的病历数据管理能力。

### 1.2 项目目标

| 目标 | 说明 |
|------|------|
| **最小可行产品 (MVP)** | 快速启动核心功能 |
| **RBAC 权限管理** | 基于角色的访问控制 |
| **数据安全** | JWT 认证 + Spring Security |
| **可扩展性** | 模块化设计，便于扩展业务 |

### 1.3 技术选型依据

- **Spring Boot 3.2**: 最新 LTS 版本，支持 Java 17+，性能优化
- **PostgreSQL**: 功能强大的开源关系数据库，医疗数据存储的理想选择
- **MinIO**: 高性能对象存储，替代阿里云 OSS，支持私有化部署
- **Redis**: 高性能缓存，用于 Token 存储和权限缓存
- **预留 APIJSON**: 支持后续零代码 API 扩展

### 4.2 请求处理流程

```
1. 请求到达 → CORS Filter (跨域处理)
                    ↓
2. JWT Filter → 从 Header 提取 Token
                    ↓
3. 验证 Token → 有效: 注入用户信息到 SecurityContext
              → 无效: 返回 401
                    ↓
4. 权限检查 → @PreAuthorize 或 配置的 URL 权限
                    ↓
5. 进入 Controller → 处理业务逻辑
                    ↓
6. 返回统一 Result → JSON 响应
```

## 5. RBAC 权限模型

### 5.1 模型说明

本项目采用 **RBAC (Role-Based Access Control)** 基于角色的访问控制模型：

```
用户 (User) ──┬── 用户角色关联 (UserRole) ──┬── 角色 (Role)
              │                              │
              │                              ▼
              │                         角色菜单关联
              │                        (RoleMenu)
              │                              │
              ▼                              ▼
         可登录系统                    菜单/权限 (Menu)
                                       - 目录 (type=1)
                                       - 菜单 (type=2)
                                       - 按钮 (type=3)
```

### 5.2 权限控制流程

```java
// 1. 登录时获取权限列表
List<String> permissions = sysUserService.getUserPermissions(userId);
// 返回: ["system:user:list", "system:user:add", "system:role:list", ...]

// 2. 权限存入 SecurityContext
List<SimpleGrantedAuthority> authorities = permissions.stream()
    .map(SimpleGrantedAuthority::new)
    .collect(Collectors.toList());

// 3. 接口使用 @PreAuthorize 控制
@PreAuthorize("hasAuthority('system:user:add')")
public Result<Void> addUser(...) { }
```

### 5.3 权限标识规范

```
格式: 模块:资源:操作

示例:
- system:user:list    # 系统模块-用户资源-查看操作
- system:user:add     # 系统模块-用户资源-添加操作
- system:role:delete  # 系统模块-角色资源-删除操作
- emr:record:view     # EMR模块-病历资源-查看操作
```

## 10. 二次开发指南

### 10.1 添加新业务模块

1. 在 `modules/` 下创建新目录，如 `modules/emr/`
2. 创建标准结构:
   ```
   modules/emr/
   ├── controller/
   ├── service/
   │   └── impl/
   ├── mapper/
   ├── entity/
   └── dto/
   ```
3. 在数据库中添加对应的表
4. 在 `sys_menu` 中添加菜单和权限配置

### 10.2 添加新接口权限

```java
// 1. 在 Controller 方法上添加权限注解
@PreAuthorize("hasAuthority('emr:record:add')")
@PostMapping("/record")
public Result<Void> addRecord(...) { }

// 2. 在数据库 sys_menu 表中添加对应权限
INSERT INTO sys_menu (parent_id, name, type, perms, ...)
VALUES (父菜单ID, '添加病历', 3, 'emr:record:add', ...);

// 3. 给角色分配该权限
INSERT INTO sys_role_menu (role_id, menu_id) VALUES (角色ID, 新菜单ID);
```

## 11. 常见问题

### Q1: 登录返回 401?

检查:
1. 用户名密码是否正确
2. 数据库初始化脚本是否执行
3. 用户状态是否为 1 (启用)

### Q2: 文件上传失败?

检查:
1. MinIO 服务是否启动
2. `application-dev.yml` 中 MinIO 配置是否正确
3. 存储桶是否存在

### Q3: Redis 连接失败?

检查:
1. Redis 容器是否启动
2. 端口 6379 是否被占用
